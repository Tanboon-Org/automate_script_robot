GitHub Actions — ค่าใช้จ่ายและ Limit ของ Tanboon-Org/automate_script_robot

จัดทำ: 2026-07-06 · ตรวจสถานะ repo จาก GitHub API ณ วันจัดทำ

✅ สรุปสั้น: repo นี้เป็น public — รันบน GitHub-hosted standard runner ได้ฟรี ไม่จำกัดนาที ไม่นับ storage ของ artifact และไม่มีทางถูกหักเงิน

ทำไมถึงฟรี

GitHub ให้ repo แบบ public ใช้ standard runner (ubuntu-latest, windows-latest, macos-latest) โดยไม่คิดค่านาทีและไม่คิดพื้นที่เก็บ artifact ทั้งสอง workflow ของ project นี้ (robot.yml, quality-scan.yml) ใช้ ubuntu-latest ธรรมดา จึงเข้าเกณฑ์ฟรีทั้งคู่

Limit ที่ยังมีอยู่ (แม้จะฟรี)

เรื่องLimitสถานะของ project นี้
เวลารันต่อ job สูงสุด 6 ชั่วโมง/job ปลอดภัย ตั้ง timeout-minutes ไว้ 45 (robot) / 30 (quality-scan) นาที
Job รันพร้อมกัน 20 jobs (Free plan) ปลอดภัย มีแค่ 2 workflows และมี concurrency group กันรันซ้อนอยู่แล้ว
Scheduled cron ถูกปิดอัตโนมัติเมื่อ repo ไม่มี activity ติดต่อกัน 60 วัน ระวัง quality-scan.yml รัน cron ทุกวัน 03:00 (เวลาไทย) — ถ้าหยุด push นาน 60 วัน cron จะเงียบไปเอง ต้องเข้าไปกด enable ใหม่ในแท็บ Actions
Runner ขนาดใหญ่ (larger runners เช่น 8-core, GPU) ไม่ฟรี แม้ repo จะ public ปลอดภัย ใช้ standard runner เท่านั้น
Workflow run ทั้ง run สูงสุด 35 วัน ปลอดภัย ไม่เกี่ยวกับ use case นี้

วิธีดูยอดใช้ / billing

  • ระดับองค์กร: github.com/organizations/Tanboon-Org/settings/billing → หน้า Usage (ต้องเป็น org owner ถึงจะเห็น)
  • ระดับ run: เข้า repo → แท็บ Actions → คลิก run ใดก็ได้ → เมนู Usage มุมขวา จะแสดงเวลา billable — ของ public repo จะขึ้น 0 เสมอ

ถ้าวันหนึ่งเปลี่ยน repo เป็น private

ตรงนี้คือจุดที่เริ่มนับโควตา แต่ค่า default ของ GitHub ปลอดภัย ไม่หักเงินเงียบ ๆ:

Plan ขององค์กรนาทีฟรี/เดือนArtifact storage ฟรี
Free2,000 นาที500 MB
Team3,000 นาที2 GB
Enterprise50,000 นาที50 GB
  • นาทีนับตาม OS: Linux ×1, Windows ×2, macOS ×10 — ของเราเป็น Linux ล้วน
  • ประมาณการยอดใช้ของ project นี้ถ้าเป็น private: cron ทุกวัน ~30 นาที ≈ 900 นาที/เดือน + robot run ละ ~45 นาทีต่อ push/PR เข้า main → มีโอกาสเกิน 2,000 นาที (Free plan) ได้ไม่ยาก
  • โควตาหมดแล้วเกิดอะไรขึ้น: spending limit ของ Actions ตั้งต้นเป็น $0 — job จะรันไม่ขึ้น (fail ตอน queue) จนกว่าจะขึ้นรอบเดือนใหม่ จะเสียเงินก็ต่อเมื่อ org owner ผูกบัตรและปรับ spending limit ขึ้นเองเท่านั้น ไม่มีการหักเงินอัตโนมัติโดยไม่ได้ตั้งใจ
  • Artifact ที่ workflow เก็บไว้ 14 วัน (โฟลเดอร์ results/ — log.html, output.xml, screenshots) จะเริ่มนับ storage ด้วยเมื่อเป็น private
⚠️ เรื่องที่สำคัญกว่าค่าใช้จ่าย: repo เป็น public + มี credentials หลุดอยู่ในโค้ด การเป็น public ทำให้ finding ข้อ 1 ในรายงานรีวิว (docs/QA-Project-Review-2026-07-06.md) หนักขึ้นมาก — password และ webUserToken ที่ commit อยู่ใน resources/variables/test_data/users.json และ wnw-checkout-orderflow.testdata.json ถูกใครก็ได้เปิดดู รวมถึง bot ที่ scan หา credentials บน GitHub ตลอดเวลา → ควร rotate token/password ฝั่ง backend โดยเร็วที่สุด แล้วย้ายค่าไปอ่านจาก GitHub Secrets / env var
